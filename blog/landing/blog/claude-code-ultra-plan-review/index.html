Claude Code Ultra Plan2.0？计划模式进化了？

Claude Code 这2天更新了一个叫 Ultra Plan 的隐藏功能，翻译过来就是「超级计划模式」。

和普通的计划模式不同，它不再局限于本地终端——它会把你的计划发送到云端，由多个跑着 Opus 4.6 的智能体并行工作，帮你构建出一份更深入、结构更完善的计划。

我用同一个提示词，分别在普通计划模式和超级计划模式下各跑了一遍。下面是我的真实体验。

怎么用

很简单，在 Claude Code 里输入 /ultraplan，后面跟上你的提示词，回车就行了。

它会问你是否发送到云端，确认之后会给你一个链接，复制到浏览器里打开，就能看到云端正在帮你规划计划了。

它是怎么运作的

整个流程分四步：

  1. 在本地终端的 Claude Code 里发起超级计划模式
  2. Claude Code 把任务发送到云端，多个智能体并行处理计划
  3. 你在浏览器里审查计划——可以加评论、提修改意见，觉得不对就让它改
  4. 确认没问题后，把计划发回终端，由本地开始执行

有个细节值得一提：云端在规划的时候，你的终端是空闲的，可以继续做别的事。

速度对比

我用 Pro 账号做了对比：

超级计划模式普通计划模式
计划规划话都没说几句就写完了4 分 40 秒
计划执行3 分 20 秒7 分 31 秒

超级计划模式在速度上有明显优势。我觉得不只是规划快，执行也快了——可能是因为计划本身写得更完善，执行时能更精准地按指令走。

Token 消耗

速度快是有代价的。

  • 超级计划模式：吃掉了 Pro 账号 5 小时限额的 70%
  • 普通计划模式：只消耗了 37%

差不多是两倍的消耗。要知道我这个提示词写得还算完善，实际改动的地方并不多，它依然跑了 70% 的额度。推测是云端多个智能体并行工作的时候，每个都在烧 token。

四个你必须知道的限制

1. 只能在终端里用

桌面版、网页版、IDE 插件统统不行。唯一的例外是在 IDE 里打开终端，从终端启动 Claude Code。

2. 必须有 Git 仓库

因为它要和云端同步代码，所以你的项目必须有一个 Git 仓库。不用怕，这个操作很简单——随便建个文件夹，在里面启动 Claude Code，让它帮你初始化一个 Git 仓库就行了。

3. Skill 调用会失灵

这是我测试中遇到的最头疼的问题。在云端规划计划的时候，Skill 会被忽略，不会主动调用。

目前我找到的唯一解决办法：在浏览器审查计划的时候，手动在评论里写上你要调用的 Skill 名称，强制它去执行。有点像在终端里用 / 加 Skill 名称强制触发的感觉。

官方文档里关于 Skill 有一些说明，但关于超级计划模式的文档本身就少得可怜，不确定是还没写完还是在调试中。

4. 云端执行可能无法同步

我在测试中发现，如果选择直接在云端执行计划（而不是发回终端执行），结果似乎没有同步到我的 Git 仓库。也就是说，虽然云端写了计划、也执行了，但本地什么都没变。

我不确定这是我的网络问题还是功能本身的限制。

顺带说一句，普通计划模式跑出来的效果其实也不差。比如它做的页面有一些细节处理——鼠标悬浮会有高亮效果，后面的示例有类似 Obsidian 双向链接的感觉，比超级模式那个浮动效果反而更好。

所以并不是说超级模式一定比普通模式强。

我的结论

这个功能是在前几天的源代码泄漏中被发现的，给人的感觉是匆匆推出来的。官方文档做得云里雾里，连它应该在什么场景下使用都不太明确。

就我目前测试下来：

  • 唯一明确的优点就是速度快
  • Token 消耗是普通模式的将近两倍
  • Skill 调用不稳定，某些情况下效果还不如普通计划模式

我手上没有特别大型的任务，所以我没法验证它在复杂项目下的表现。也许 Ultra Plan 就是为大型项目准备的——如果你手上正好有大项目，可以拿它跑一下，看看和普通模式的差别到底在哪。

但就目前来说，为了省额度、求稳定，普通计划模式对大多数人来说够用了。
