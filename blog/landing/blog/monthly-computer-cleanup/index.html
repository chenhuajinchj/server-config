效率 文件管理 AI Agent Obsidian

从 Notion 重度用户到让 AI 帮我整理电脑

整理的意义不在结果，而在过程本身。分享我的文件管理方式：简化分类、Git 状态管理、iCloud 软连接、归档替代删除。

我建议每个人每个月都要整理一下自己的电脑，看一下自己平时存的文件和文档，有哪些内容是需要整理的，哪些是自己应该保留下来的，哪些是需要完全清除的。

整理的意义不是在过程

这个过程的意义并不在于整理本身，或者说不在于最终的结果，而是在于"整理"这一个动作的本身。通过整理，你可以看到：

  1. 之前写的文档和做的项目。
  2. 哪些地方存了一些你保存下来但从未打开过，或者保存了却没看过的文件。

你可以在整理的过程中顺便把它们看一看。也可以在整理的过程中，把你之前写完发布的文章拿出来翻一翻。

从一个月后去看一个月前写的内容，你的心情是怎样的？你觉得还有什么进步的空间？我觉得这些才是整理的真正意义。

复杂系统是累赘，只是让自己满意

很多人喜欢给自己搭一个臃肿的系统，包括之前的我自己。

比如我之前在 Notion 里面花费了将近半个月的时间，搭建了一套完全自我的工作库。

当时搭建完成之后，我觉得自己简直是"天下第一"，觉得有了这套工作流程、工作库和知识库，我就可以产出更多更好的内容，可以坚持不懈地完成打卡。

但是现在，尤其是在我和 Claude Code（Anthropic 推出的 AI 编程工具，下面简称 CC）的交流与工作融合越来越多之后，我发现这种东西对于 AI Agent 来说，路径越多，造成的干扰其实就越多。

同时，对于我们人类而言，也是因为每一次打开的路径越长，我们做一件事的摩擦力就越大，那么这件事情就越做不下去了。

我现在仅仅只做了一个 macOS 自动操作（Automator）脚本，根据下载的文件种类自动帮我移动到对应的文件夹中，其他的我都不会去做。平常的一些文档我都是随开随看随存，看完就关。

Agent 时代的文件管理

我现在觉得最好的方式是：

  1. 简化你的分类：不管是代码还是日常文档，只归为一个大类。
  2. 细分你的项目：在大类里面，再根据具体的项目去分开创建。
  3. 状态管理：每一个文件、每一个项目都应该用 Git 管理。这样你既可以回滚之前的状态，也不用担心哪天手误删掉了就找不回来。

你不要觉得这只是方便了 Agent，当你和越来越多的 AI 工具协作之后会发现：对 Agent 而言最高效的方式——路径短、项目明确——其实对人也是最高效的。

我觉得与其花时间去打造一些华而不实、分门别类的知识库或个人管理库，还不如把时间花在真正的产出上面。我现在已经不再搭建复杂的系统了，只是每个月定期清理一次。

比如今天是 4 月 2 号，本来昨天就该清理，我在每个月的第一天整理一次，但由于昨天没有清理，所以一直拖到了今天。

我的 Obsidian 同步方案：iCloud + 软连接

在这个过程中，分享一点我自己的 Obsidian 同步和保存方式。

我是通过苹果的 iCloud 云文档来同步的，这样我可以在任何一台手机、iPad 或电脑上直接同步使用。我不会去用 Git 来同步 Obsidian 的笔记，因为那会增加我的操作摩擦。

但 iCloud 有一个问题：默认路径非常长。很多时候 AI Agent 应用（比如 Codex 或 CC）在你和它对话时，如果你让它帮你在 Obsidian 里面创建或寻找某个文档，它需要花费很多时间去寻找，尤其是在你没有给它指定路径的情况下。

虽然这是一个很基础的知识，但很多时候我们会不自觉地默认 AI 已经知道路径在哪，结果导致它浪费了很多上下文、时间还有 Token。

所以，我做了一个这样的操作：通过"软连接"的方式，把 Obsidian 的路径映射出来。这样我可以直接在终端输入 cd ~/Obsidian 就能连接到我的库，而原本的文件依然存放在 iCloud 云文档里。

关于软连接的实现方式，你可以让任何一个 AI Agent 应用（比如 Codex 或 CC）来帮你完成，一句话的事。

用归档替代废纸篓

最后分享一个小习惯：整理文件时，很多人会把目前看起来不需要的东西直接删除或移到废纸篓。

我的建议是建立一个"归档"文件夹。如果你不确定某些东西未来是否还有用，就先放进归档。到了下个月再整理时，优先处理归档文件夹里的内容——如果确定不需要了，再删除。

这和废纸篓的区别在于：归档是存放那些你已经用不到、但又不确定未来会不会需要的文档、文件和项目，在你下个月进行清理的时候，你就可以回看上一个月你放到"归档"里面的内容。你可以很轻易地判断出来，哪些东西是你不需要了，哪些东西是你应该保留下来的。

而且这种结构对 Agent 特别友好——"归档"这个语义本身就是清晰的，Agent 可以很好地理解哪些内容是活跃的、哪些是归档的，不需要你额外解释。

我的整理是完全针对我电脑上的所有文件，而不仅仅只是局限于 Obsidian。
